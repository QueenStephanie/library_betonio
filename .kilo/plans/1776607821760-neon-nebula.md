# Librarian Portal Redesign Plan (Borrower Theme Alignment)

## Objective
Redesign the **librarian portal pages** to match the borrower-side visual theme (hero sections, stat cards, surface cards, spacing rhythm, buttons, alerts, table styling), while preserving current librarian workflows and backend behavior.

## Confirmed Scope
- Include: `app/librarian/index.php`, `app/librarian/books.php`, `app/librarian/circulation.php`, `app/librarian/reservations.php`, `app/librarian/fines.php`
- Exclude: librarian/admin auth/profile pages outside the portal flow
- Keep: existing role checks, CSRF flow, form actions, repository calls, and page logic

## Current State Findings
- Borrower pages already implement a cohesive shell (`borrower-page`, `borrower-shell`, hero + stat + card layout) using `public/css/borrower.css`.
- Librarian pages currently use generic `admin.css` styles and less consistent page composition.
- Sidebar is shared via `app/shared/portal-sidebar.php` and should remain unchanged structurally.

## Implementation Plan
1. **Create a dedicated librarian theme layer that mirrors borrower visual language**
   - Add `public/css/librarian.css` with librarian-specific classes patterned after borrower composition:
     - page/shell wrappers
     - hero blocks
     - stat tiles
     - panel headers/content
     - action button groups
     - responsive table wrappers and empty states
   - Reuse existing color direction from admin palette, but match borrower-style gradients, depth, border radii, and spacing cadence.
   - Verification: CSS file loads without syntax errors and introduces no global selector collisions (scope with `.librarian-*` and `.portal-role-librarian`).

2. **Refactor shared librarian page scaffolding to the new layout primitives**
   - For each page (`index.php`, `books.php`, `circulation.php`, `reservations.php`, `fines.php`):
     - Wrap content in `librarian-page` + `librarian-shell`
     - Replace basic header blocks with borrower-like hero sections
     - Normalize section cards to a common visual structure
   - Keep all existing PHP data preparation and controls unchanged.
   - Verification: each page renders with same data and actions, but new layout hierarchy is present in markup.

3. **Apply consistent component patterns across all librarian pages**
   - Standardize:
     - stats rows/cards
     - alert blocks
     - table panel layout
     - action button visual treatment
     - filter/search sections
   - Ensure modals in `books.php` inherit redesigned visual style without changing behavior.
   - Verification: visual consistency across all 5 pages; no missing styles on existing controls.

4. **Wire stylesheet and role-safe class hooks**
   - Update the 5 librarian PHP pages to include `public/css/librarian.css` after `admin.css` so overrides are deterministic.
   - Add/adjust page-level wrapper classes only where needed; avoid touching unrelated borrower/admin pages.
   - Verification: CSS load order confirmed; no borrower page regressions.

5. **Responsive and interaction validation**
   - Validate desktop/tablet/mobile behavior for:
     - sidebar + main area composition
     - hero collapse rules
     - table overflow handling
     - button/form spacing
   - Verification: no clipped content, overlapping elements, or unusable controls at common breakpoints.

6. **Functional regression checks (behavior unchanged)**
   - Run backend tests: `php backend/tests/run.php`
   - Manually smoke-check librarian actions:
     - add book modal open/close + submit
     - checkout/checkin actions
     - reservation approve/reject/cancel/checkout
     - fines table rendering
   - Verification: existing workflows still execute successfully with redesigned UI.

## Files Expected To Change
- `public/css/librarian.css` (new)
- `app/librarian/index.php`
- `app/librarian/books.php`
- `app/librarian/circulation.php`
- `app/librarian/reservations.php`
- `app/librarian/fines.php`

## Acceptance Criteria
- All librarian portal pages share a borrower-inspired visual system.
- No functional regressions in librarian actions or permissions.
- Mobile and desktop layouts are usable and consistent.
- Redesign is scoped to librarian portal pages only.

## Risk Controls
- Keep all server-side logic and form handlers intact; touch mostly markup classes and stylesheet includes.
- Scope styles to `.portal-role-librarian` and `.librarian-*` to avoid bleed into borrower/admin-auth screens.
- Preserve existing IDs/names used by JS and POST handlers to avoid breaking interactions.
